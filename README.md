# ITISDEVS02-Group3

	Speaktrum is a specialized mood board system designed to bridge the communication gap for children on the autism spectrum. By replacing verbal questioning with a friendly digital interface, Speaktrum empowers children to express complex emotions and identify specific sensory triggers. The application makes the emotional check-in process easier through a tap and log system or the camera feature to ensure that even non-verbal or overstimulated users can still communicate their state without the cognitive exhaustion of speech. The project integrates real-time tracking and history insights to support collaborative care amongst children with special needs. By acting as a centralized system, Speaktrum allows parents, teachers, and healthcare specialists to share data and synchronize their support strategies based on the child’s actual needs. 

# Code Overview
The system is built using the MVC (Model-View-Controller) architectural pattern with Node.js and Express.js stack. The codebase is organized into separate folders wherein each layer has one clear responsibility. Models will handle data, controllers will handle logic, while views will handle the display, and utilities will handle specialized tools.

The file app.js is the entry point and root of the system. It is responsible for establishing a connection with the database MongoDB, registering all Express middleware such as body parsing, static file serving, and method override support, mounting the route definitions, setting EJS as the templating engine, and starting the daily email reminder scheduler. Every request made to the system passes through this file first before being directed to the appropriate route.

The file models/Mood.js represents the Model layer of the MVC. This is where the definition of the MongoDB document schema using Mongoose, which enforces the structure and validation rules for every mood record stored in the database. It manages the child’s name stored as a string, the emotion as one of the six basic emotions, the intensity as a number between 1 and 10, the trigger description as a string, and the date and time the emotion occurred. This file ensures that no incomplete or invalid data can enter the database. 

The folder and file controllers/moodController.js is the controller layer and serves as the business logic center of the system. This is where eight action functions handle CRUD operations. The controller also contains helper functions, such as getAlertLevel(), which classifies each entry by intensity severity for the escalation system, and buildHeatmap(), which maps emotion frequency across the past seven days to produce the heatmap visualization data. 

routes/moodRoutes.js is the URL routing layer and traffic director of the system. It maps HTTP methods and URL to their corresponding controller functions following RESTful conventions. This file contains no business logic of its own - its sole responsibility is to receive an incoming request and forward it to the correct controller action, keeping the routing layer clean and easy to read.

The file and folder utils/mailer.js is an IT tool that implements an automated daily email reminder system using Nodemailer with Gmail as the transport service. When the server starts, it launches a background scheduler that checks every minute whether the current hour matches the configured reminder time, which defaults to eight in the evening. If the target hour is reached and no mood entry has been logged for that day, the system queries MongoDB to confirm and then dispatches an HTML-formatted reminder email to the parent’s email address. Furthermore, it also sends a report to the parent at the end of the week.

public/js/camera.js is another specialized IT tool that utilizes real-time facial emotion recognition using face-api.js, a machine learning library built on TensorFlow.js. When the child clicks the camera button on the mood logging form, the script requests access to the device’s camera. It then loads two pre-trained neural network models from the local public/models/folder and begins a detection loop that runs once per second. When an emotion is identified with a confidence level above 60%, the script automatically highlights the matching emotion button on the form with a pulsing animation, allowing the child’s emotional state to be suggested hands-free before they confirm their selection.

The folder and file public/css/style.css contains all visual styling for the entire application. It utilizes CSS custome properties for consistent theming across every page and covers all UI components including the navigation bar, mood timeline cards, the child-friendly emotion picker grid, the intensity slider, the trigger chip selector, the report charts, the heatmap grid, the camera panel, and responsive layout adjustments for smaller screens. 

public/js/main.js is a client-side utility script with a single responsibility. It highlights the active link in the navigation bar based on the current page URL, giving the user a clear visual indicator of where they are in the application.
